План архітектури для проекту       інтернет-магазину «Vervilure»

1. Загальна архітектура системи
1.1. Архітектурний стиль
Для інтернет-магазину Vervilure застосовується монолітна архітектура з використанням фреймворку Django. Цей вибір обґрунтований наступними факторами:
    • Django надає єдину, узгоджену систему, яка добре підходить для e-commerce проектів 
    • Адмін-панель: Django має потужну вбудовану адмін-панель, вона буде використовуватися для управління контентом, товарами, замовленнями
    • Монолітна архітектура спрощує початковий розвиток і розгортання
    • Проектом легше керувати на початкових етапах
1.2. Патерни проектування
Рекомендую використати такі патерни:
    • MVC (MVT в Django) - для відокремлення бізнес-логіки від представлення 
    • Repository - для абстрагування доступу до даних 
    • Factory - для створення складних об'єктів (наприклад, замовлень) 
    • Observer - для реалізації системи сповіщень

3. Модульна структура
3.1. Модуль "Користувачі та аутентифікація" (accounts)
    • Реєстрація через email або соцмережу (будь-яку на вибір розробників)
    • Вхід у систему
    • Відновлення пароля
    • Перегляд історії замовлень
    • Управління збереженими товарами
    • Редагування особистих даних
3.2. Модуль "Каталог товарів" (catalog)
    • Опис:
Каталог дозволяє користувачам переглядати, шукати та фільтрувати товари.
Функціональні елементи:
    • Відображення товарів
        ◦ Картки товарів: фото, назва, ціна, рейтинг, короткий опис
    • Фільтрація
        ◦ За категоріями (вази, картини, текстиль, світильники тощо)
        ◦ За ціною (повзунок мінімум/максимум)
        ◦ За матеріалами (кераміка, дерево, скло, текстиль)
    • Пошук
        ◦ Повнотекстовий пошук із підказками
    • Детальна сторінка товару
    • Галерея фото
    • Розширений опис (матеріали, історія створення, догляд)
    • Відгуки покупців
    • Рейтинг
    • Динамічне оновлення ціни за опціями (розмір, колір)
    • Рекомендовані товари
    • Кнопки «Додати в кошик» та «Додати у вішліст»
3.3. Модуль "Кошик" (cart)
    • Опис:
Кошик дозволяє користувачам переглядати та редагувати свій вибір перед оформленням замовлення.
Функціональні елементи:
    • Відображення списку товарів із можливістю змінювати кількість
    • Видалення товарів із кошика
    • Автоматичний перерахунок загальної вартості
    • Поле введення промокоду
    • Вибір способу доставки
    • Вибір способу оплати (банківська картка, PayPal, Apple Pay, Google Pay. Одне на вибір розробників)
    • Підтвердження замовлення
    • Надсилання електронного листа з деталями покупки
3.4. Модуль "Замовлення" (orders)
    • Оформлення замовлення 
    • Управління статусами 
    • Інтеграція з платіжними системами 
3.5. Модуль "Вішліст" (wishlist)
    • Опис:
Вішліст дозволяє користувачам зберігати товари для подальшого перегляду чи покупки.
Функціональні елементи:
    • Додавання товару до вішлісту
    • Видалення товару з вішлісту
    • Перегляд списку збережених товарів у кабінеті користувача
    • Можливість перенести товар із вішлісту в кошик
    • 
3.6. Модуль "Відгуки" (reviews)
    • Опис:
Відгуки формують довіру до товарів і бренду.
Функціональні елементи:
    • Залишення відгуку (тільки після покупки)
    • Додавання фото до відгуку
    • Оцінка товару за 5-бальною шкалою
    • Перегляд відгуків на сторінці товару
    • 
3.7. Модуль "Програма лояльності" (loyalty)
    • Опис:
Програма лояльності мотивує повторні покупки.
Функціональні елементи:
    • Нарахування балів після кожної покупки
    • Відображення балансу в кабінеті користувача
    • Використання балів як знижки під час покупки
    • Email-сповіщення про нарахування чи використання балів

4. Потік даних
4.1. Основні бізнес-процеси
    1. Перегляд каталогу товарів:
        ◦ Користувач → Каталог → Фільтрація → Відображення товарів 
    2. Оформлення замовлення:
        ◦ Користувач → Додавання у кошик → Оформлення замовлення → Оплата → Підтвердження 
    3. Управління вішлістом:
        ◦ Користувач → Додавання у вішліст → Перегляд вішлісту → Перенесення у кошик 
    4. Програма лояльності:
        ◦ Замовлення → Нарахування балів → Використання балів при замовленні

6. API структура
Рекомендую реалізувати REST API для взаємодії фронтенду з бекендом:
/api/v1/
  /catalog/            # Каталог товарів
  /cart/               # Кошик
  /orders/             # Замовлення
  /users/              # Користувачі
  /wishlist/           # Вішліст
  /reviews/            # Відгуки
  /loyalty/            # Програма лояльності

7. Технологічний стек
    • Бекенд: Django, Django REST Framework 
    • База даних: PostgreSQL 
    • Кешування: Redis 
    • Фронтенд: HTML, CSS, JavaScript (або фреймворк на вибір) 
    • Розгортання: Docker, Docker Compose 
    • CI/CD: GitHub Actions
